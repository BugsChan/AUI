# AUI框架实现计划

## 核心组件设计

### 1. 主入口组件 (aui-main.vue)

* **功能**：管理整体状态，处理配置解析

* **实现要点**：

  * 接收开发者传入的配置参数

  * 管理icon和窗口的显示状态

  * 处理响应式布局（PC/移动端适配）

  * 集成所有子组件

### 2. 图标组件 (aui-icon.vue)

* **功能**：显示可点击的AI助手图标

* **实现要点**：

  * 支持自定义图标路径、大小、描述

  * 响应式位置布局（PC/移动端不同）

  * 点击事件处理（显示/隐藏窗口）

### 3. 窗口组件 (aui-win.vue)

* **功能**：提供交互界面

* **实现要点**：

  * PC端：可拖拽调整位置

  * 移动端：固定底部布局

  * 支持语音/文字输入

  * 显示消息历史和生成的卡片

  * 消息发送和接收逻辑

### 4. 卡片组件 (aui-card.vue)

* **功能**：根据配置动态生成功能卡片

* **实现要点**：

  * 支持多种UI类型（input、button、journey等）每个卡片顶部有一句提示词，中间有一张图片，下方可以有输入框，最后有确认按钮

  * 参数动态绑定

  * 确认按钮事件处理

  * 卡片内容渲染（包括图片和提示信息）

## 核心逻辑模块

### 1. 配置解析器

* **功能**：解析开发者传入的JSON配置

* **实现要点**：

  * 验证配置格式

  * 提取icon、window、description和methods配置

  * 初始化默认值

### 2. 消息处理器

* **功能**：处理用户输入和大模型响应

* **实现要点**：

  * 接收用户文字/语音输入

  * 模拟大模型响应（实际项目中替换为真实API调用）

  * 根据配置生成对应的卡片

### 3. 功能执行器

* **功能**：执行配置的功能

* **实现要点**：

  * 区分函数调用和超链接跳转

  * 处理参数传递

  * 执行回调函数

  * 处理超链接跳转

## 实现步骤

1. **初始化项目结构**

   * 更新main.js，创建可导入的Vue插件

   * 设计组件间的通信机制

2. **实现核心组件**

   * 开发aui-icon.vue，实现响应式图标

   * 开发aui-win.vue，实现基础窗口布局

   * 开发aui-card.vue，实现卡片动态生成

   * 整合到aui-main.vue

3. **实现配置系统**

   * 开发配置解析器

   * 实现功能配置的存储和管理

4. **实现交互逻辑**

   * 添加窗口拖拽功能（PC端）

   * 实现消息输入和显示

   * 添加卡片生成和点击事件

5. **实现功能执行**

   * 处理函数调用

   * 处理超链接跳转

   * 实现参数传递

6. **响应式适配**

   * 添加PC/移动端布局适配

   * 优化移动端体验

7. **测试和文档**

   * 编写示例代码

   * 测试各项功能

   * 完善文档

## 技术栈

* Vue 3

* JavaScript

* CSS3（响应式布局、拖拽效果）

* Web Speech API（语音输入，可选）

## 预期效果

开发者引入AUI框架后，只需通过JSON配置即可快速集成AI助手功能，网站使用者可以通过点击图标与AI助手交互，获取网站相关信息或执行预设功能。
